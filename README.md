# Dallas ISD EdTech Spending Analyzer

A data pipeline and reporting tool for analyzing educational technology vendor spending in Dallas Independent School District, using four years of publicly available check register data (FY2022–FY2025).

**Key finding:** Dallas ISD paid $193.6 million to 24 identified EdTech vendors over four fiscal years. 78% of that spending went to vendors rated low, very low, or none on replaceability — most of it to curriculum platforms deeply embedded in district operations. 38.7% was funded through ESSER emergency relief dollars.

---

## What This Does

1. **Downloads** monthly check register PDFs from Dallas ISD's public website (48 PDFs, FY2022–FY2025)
2. **Extracts** all transactions into a flat CSV (~122,000 rows, $5.96B total spend across 9,370 vendors)
3. **Classifies** vendors using an AI-assisted two-pass research pipeline (Claude API), scoring each on five replaceability dimensions
4. **Scrapes** the Dallas ISD procurement portal for contract award data (category codes, effective/end dates)
5. **Enriches** vendor records by joining contract awards to transactions and breaking down spending by Texas fund code
6. **Generates** an interactive HTML report with charts, vendor profiles, fund breakdowns, and contract award details

## Setup

```bash
# Clone and install dependencies
git clone <repo-url>
cd check-register-analyzer
pip install -r requirements.txt

# Install Playwright browser (required for contract scraping)
playwright install chromium
```

### Generating the Report Without Re-running the Full Pipeline

The enriched data file (`data/vendors/edtech_award_spend_v1.json`) is committed to the repo. If you just want to regenerate the HTML report:

```bash
python generate_report.py
# Output: reports/web/edtech_spending_report.html
open reports/web/edtech_spending_report.html
```

## Full Pipeline (Reproducing from Scratch)

Run these steps in order. Steps 1–3 produce large files that are gitignored; the outputs of steps 4–6 are committed.

### Step 1 — Download PDFs

```bash
python -m src.pdf_downloader
# Output: data/raw/*.pdf (48 files)
```

### Step 2 — Extract Transactions

```bash
python -m src.run_extraction
# Output: data/extracted/all_transactions_raw.csv (~122,000 rows)
```

### Step 3 — AI Vendor Research

Requires an [Anthropic API key](https://console.anthropic.com/).

```bash
export ANTHROPIC_API_KEY="your-key-here"
python -m src.run_pass2_research
# Output: data/vendors/edtech_research_v2.json
```

### Step 4 — Scrape Contract Awards

```bash
python -m src.contract_scraper
# Output: data/vendors/contract_awards_raw.json
```

The scraper uses Playwright to navigate the Dallas ISD procurement portal. If any vendors fail to match automatically, place corrected JSON files in `data/manual_script_extraction/` and merge them:

```bash
python -m src.merge_vendor_json --source data/manual_script_extraction
```

### Step 5 — Enrich

```bash
python -m src.contract_enrichment
# Output: data/vendors/edtech_award_spend_v1.json
```

### Step 6 — Generate Report

```bash
python generate_report.py
# Output: reports/web/edtech_spending_report.html

# For PDF output (requires weasyprint):
python generate_report.py --format pdf
```

## Replaceability Scoring

Each vendor is scored on five dimensions (1 = very hard to replace, 5 = easy):

| Dimension | Description |
|---|---|
| Technical Buildability | Could the district build or configure this in-house? |
| Content / IP | Is the core value in proprietary content that can't be replicated? |
| Switching Cost | How disruptive would migration away be? |
| Market Alternatives | Are there comparable competing products? |
| Data Portability | How easily can student data and records be exported? |

Scores are weighted into a composite and mapped to levels: `high`, `medium`, `low`, `very_low`, `none`.

## Fund Code System

Texas school districts use a standardized chart of accounts under the [Financial Accountability System Resource Guide (FASRG)](https://tea.texas.gov/finance-and-grants/financial-compliance/financial-accountability-system-resource-guide). Key fund codes in this dataset:

| Code | Fund |
|---|---|
| 199 | General Fund |
| 211 | ESEA Title I, Part A |
| 281 | ESSER I |
| 282 | ESSER II/III (ARP) |
| 28C | ESSER/COVID Relief |

## Data Sources

- **Check registers:** Published monthly by Dallas ISD at `dallasisd.org`
- **Procurement awards:** Dallas ISD vendor award portal at `www2.dallasisd.org`
- **Vendor research:** Generated via Anthropic Claude API; see `src/vendor_research.py` for prompts

---

## Limitations

- The procurement portal only surfaces currently active or recently expired contracts. Prior award cycles that have rolled off the site are not captured — 6 of 24 vendors show spending with no matching current award.
- 6 vendors ($20.9M in spending) could not be located in the procurement portal at all, possibly operating under parent company names or cooperative purchasing agreements.
- All 33 matched contract awards are `ongoing` (extending past August 2025). The portal does not appear to retain completed historical contracts for these vendors.
- Fund code attribution reflects which fund issued each check, not the ultimate source of the funds.
- The 24 EdTech vendors identified represent the most significant and recognizable relationships; the true EdTech total across all ~9,370 vendors is likely higher.
